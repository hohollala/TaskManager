Shrimp Task Manager

문제점을 해결하고 효율적인 작업 흐름을 제공하는 지능형 시스템

장기간의 작업 기억력을 가진 AI 어시스턴트를 제공하여 복잡한 작업을 구조화하고 실행 추적을 제공하여 효율적인 작업 경험을 제공합니다.

문제점과 해결 솔루션

Shrimp Task Manager는 AI 프로그래밍 어시스턴트를 위해 설계된 구조화된 작업 관리 시스템으로, 지능형 작업 계획, 자동 분해, 실행 추적 및 완성도 검증 기능을 제공합니다.

기억 부족

AI 어시스턴트는 대화 간 작업 기억력이 부족하여 장기간 작업 진행 상황을 추적할 수 없고, 동일한 요구사항을 반복적으로 설명하여 시간과 자원을 낭비합니다.

작업 기억 기능

자동으로 실행 이력을 저장하여 장기간의 기억력을 제공하여 AI 어시스턴트가 이전 작업 진행 상황을 기억할 수 있습니다.

구조 혼란

복잡한 작업에 대한 체계적인 관리가 부족하여 효율성이 떨어지고, 하위 작업 간의 의존 관계 관리가 부족하여 전체 진행 상황을 추적하기 어렵습니다.

구조화된 작업 분해

복잡한 작업을 관리 가능한 하위 작업으로 분해하여 명확한 의존 관계를 설정하고 순차적인 실행 경로를 제공하여 효율적으로 완성할 수 있습니다.

반복 작업

과거의 경험과 해결책을 효과적으로 활용할 수 없고, 매번 대화를 통해 새로운 요구사항을 처음부터 다시 설명해야 합니다.

지식 누적과 경험 참고

성공적인 해결책을 자동으로 기록하여 작업 지식 데이터베이스를 구축하고, 유사한 작업에 대한 빠른 참고를 통해 경험을 누적하고 지식을 재사용할 수 있습니다.

핵심 기능 탐구

핵심 기능

Shrimp Task Manager는 6가지 핵심 기능을 제공하여 복잡한 작업을 효율적으로 관리하고 실행하며 추적할 수 있습니다.

스마트 작업 계획과 분석

요구사항과 제약 조건을 깊이 분석하여 구조화된 작업 계획을 생성합니다. 자동으로 범위, 위험 및 우선순위를 평가하여 이성적이고 종합적인 실행 전략을 제공합니다.

자동 작업 분해와 의존 관리

복잡한 작업을 관리 가능한 작은 작업으로 분해하고 작업 간의 의존 관계를 식별하여 최적화된 실행 경로를 설정하여 자원 충돌과 실행 병목을 방지합니다.

실행 상태 추적

각 작업의 실행 상태를 실시간으로 모니터링하고 진행 상황을 시각화하여 자동으로 의존 항목 상태를 업데이트하며 작업이 완료될 때 상세한 실행 보고서를 제공합니다.

작업 완성도 검증

모든 요구사항과 기준을 철저하게 검사하여 완료되었는지 확인하고 검증 보고서와 품질 평가를 제공하여 산출물이 예상 요구사항을 충족하는지 확인합니다.

작업 복잡도 평가

다차원 기준에 기반한 작업 복잡도를 평가하여 자원 요구량을 식별하고 고위험 구성 요소를 식별하여 자원과 시간을 합리적으로 배분할 수 있습니다.

작업 기억 기능

대화 간 작업 기억력을 제공하고 실행 이력과 컨텍스트를 자동으로 저장하여 필요할 때 작업을 복원하고 계속해서 실행할 수 있습니다.

사용 예제

Shrimp Task Manager의 주요 명령어와 사용 방법을 예제를 통해 알아보세요.

새 프로젝트 생성 (new 명령어)

new 명령어를 사용하여 새로운 프로젝트 요구사항을 생성할 수 있습니다. 이 명령어는 프로젝트의 목적, 필수 기능, 디자인 요구사항 등을 포함한 상세한 요구사항을 생성합니다.

사용 방법:

new "앱 목적: 할일 관리, 필수 기능: 사용자 로그인/데이터 저장, 디자인: 기본 디자인, 서버: 새로 개발, 외부 서비스: 푸시 알림, 플랫폼: iOS/Android, 기술 스택: React Native, 기타: 없음"

생성되는 내용:

  • 프로젝트 개요 및 목표
  • 기술 스택 및 아키텍처
  • 필수 기능 목록
  • UI/UX 요구사항
  • 데이터베이스 설계
  • API 엔드포인트
  • 보안 요구사항
  • 배포 및 유지보수 계획

이 명령어를 통해 프로젝트의 전체적인 요구사항을 체계적으로 정의하고, 이후 작업 계획 및 개발 과정에서 참조할 수 있는 기반을 마련할 수 있습니다.

작업 계획 (plan 명령어)

plan 명령어를 사용하여 복잡한 작업을 계획하고 분해할 수 있습니다. 이 명령어는 작업의 요구사항을 분석하고 구조화된 계획을 생성합니다.

사용 방법:

plan "새로운 웹 애플리케이션 개발"

계획 과정:

  1. 작업 계획 - 초기화하고 작업 흐름을 상세하게 계획
  2. 깊이 분석 - 요구사항을 깊이 분석하고 기술적 실현 가능성 평가
  3. 계획 검토 - 분석 결과를 비판적으로 검토하고 계획 최적화
  4. 작업 분해 - 복잡한 작업을 관리 가능한 하위 작업으로 분해

이 방법을 통해 복잡한 대형 작업을 구조화된 작업 단위로 변환하면서 전체적인 시각을 유지할 수 있습니다.

작업 실행 (execute)

execute "task_id_here"

선택된 작업을 지정된 계획에 따라 실행합니다.

작업 목록 (list)

list

모든 작업의 목록과 현재 상태를 확인합니다.

연구 모드 (research)

research "React Native 상태 관리"

기술 주제에 대한 체계적인 연구를 수행합니다.

Prompt 사용자 정의 기능

환경 변수를 사용하여 시스템 프롬프트를 정의하여 AI 어시스턴트의 행동을 사용자 정의할 수 있습니다.

기능 개요

Prompt 사용자 정의는 환경 변수를 사용하여 AI 어시스턴트의 행동을 조정할 수 있습니다. 두 가지 사용자 정의 방식을 제공합니다: 완전히 원본 프롬프트를 대체하거나 기존 기반에 추가 내용을 추가합니다.

주요 장점

  • 개인화 제작: 특정 프로젝트나 분야에 따라 시스템 행동 조정
  • 효율성 향상: 반복되는 작업 유형에 대해 최적화하여 중복 설명 감소
  • 일관성 유지: 팀원 간 사용하는 프롬프트를 통일하여 일관된 작업 방식 보장
  • 전문성 적응성: 특정 기술 분야나 산업에 따라 전문 용어와 기준 조정
  • 팀 협업: 팀원 간 사용하는 프롬프트를 통일하여 일관된 작업 방식 보장

사용 방법

환경 변수 설정

각 기능에 대한 프롬프트를 환경 변수로 설정하여 특정 명명 규칙을 사용합니다: 

# 대체 모드: MCP_PROMPT_[FUNCTION_NAME]
# 추가 모드: MCP_PROMPT_[FUNCTION_NAME]_APPEND

# 예제:
MCP_PROMPT_PLAN_TASK="사용자 정의 작업 계획 프롬프트"
MCP_PROMPT_EXECUTE_TASK_APPEND="추가적인 작업 실행 지침"

자세한 문서를 통해 더 많은 설정 방식과 매개변수 사용 설명을 확인할 수 있습니다.

🔧 수동 설치

완전한 제어를 위한 수동 설치

개발자나 고급 사용자를 위한 완전한 수동 설치 가이드입니다. 이 방법을 통해 소스 코드를 직접 관리하고 커스터마이징할 수 있습니다.

1
저장소 클론
git clone https://github.com/hohollala/TaskManager.git
cd TaskManager
2
의존성 설치
npm install
3
프로젝트 빌드
npm run build
4
글로벌 설치
npm install -g .
5
MCP 서버 실행 테스트
node dist/index.js
⚠️ 주의사항
  • Node.js와 Git이 시스템에 설치되어 있어야 합니다.
  • 관리자 권한이 필요할 수 있습니다 (글로벌 설치 시).
  • 소스 코드를 직접 수정할 수 있어 커스터마이징이 가능합니다.
  • 업데이트 시 수동으로 git pull을 수행해야 합니다.
  • MCP 서버가 정상적으로 실행되는지 확인하세요.
🔧 환경 변수 설정 (선택사항)
Windows:
set DATA_DIR=C:\path\to\your\data
set TEMPLATES_USE=ko
set ENABLE_GUI=false
macOS/Linux:
export DATA_DIR=/path/to/your/data
export TEMPLATES_USE=ko
export ENABLE_GUI=false

Claude Code 설정

Windows PowerShell용 MCP 서버 추가:

claude mcp add shrimp-task-manager -s user -- node "$PWD/dist/index.js"

Windows CMD용 MCP 서버 추가:

claude mcp add shrimp-task-manager -s user -- node "[설치경로]/dist/index.js"

macOS/Linux용 MCP 서버 추가:

claude mcp add shrimp-task-manager -s user -- node $(pwd)/dist/index.js

수동 설정: ~/.claude.json 파일 편집

Claude Code 설정 파일을 직접 편집하여 MCP 서버를 추가할 수 있습니다.

Windows: C:\Users\사용자명\.claude.json 
{
  "mcpServers": {
    "shrimp-task-manager": {
      "type": "stdio",
      "command": "node",
      "args": [
        "[설치폴더경로]/dist/index.js"
      ],
      "env": {}
    }
  }
}

설정 방법:
1. ~/.claude.json 파일을 메모장이나 코드 에디터로 열기
2. "mcpServers" 섹션에 위의 설정 추가
3. [설치폴더경로]를 실제 설치 경로로 변경 (예: C:/claudeMCPS/TaskManager)
4. 파일 저장 후 Claude Code 재시작

Cursor IDE 설정

만약 Cursor IDE를 사용한다면 Shrimp Task Manager를 개발 환경에 통합할 수 있습니다.

전역 설정: ~/.cursor/mcp.json 
{
  "mcpServers": {
    "shrimp-task-manager": {
      "command": "node",
      "args": [
        "C:/Users/%USERNAME%/AppData/Roaming/npm/node_modules/mcp-shrimp-task-manager/dist/index.js"
      ],
      "env": {
        "DATA_DIR": "C:/Users/%USERNAME%/ShrimpData",
        "TEMPLATES_USE": "ko",
        "ENABLE_GUI": "false"
      }
    }
  }
}

프로젝트별 설정

각 프로젝트에 대해 별도 설정을 원한다면:

프로젝트별 설정: .cursor/mcp.json 
{
  "mcpServers": {
    "shrimp-task-manager": {
      "command": "node",
      "args": [
        "./node_modules/.bin/mcp-shrimp-task-manager"
      ],
      "env": {
        "DATA_DIR": "./.shrimp-data",
        "TEMPLATES_USE": "ko",
        "ENABLE_GUI": "false"
      }
    }
  }
}

중요 팁

절대 경로를 사용해야 합니다: DATA_DIR 설정을 절대 경로로 사용하고 상대 경로를 사용하지 않도록 확인하세요. 그렇지 않으면 데이터를 제대로 로드할 수 없습니다.

프롬프트 설정 설명

Shrimp Task Manager는 두 가지 모드를 지원합니다:

  • TaskPlanner: 초기 작업 계획과 복잡한 작업 분해에 적합하며, AI 어시스턴트는 작업 계획자 역할을 합니다. 
    
당신은 전문적인 작업 계획 전문가입니다. 사용자와 상호작용하여 사용자의 요구사항을 분석하고 프로젝트와 관련된 정보를 수집한 후 「plan_task」를 사용하여 작업을 만듭니다. 작업이 완료되면 요약을 작성하고 사용자에게 「TaskExecutor」 모드로 작업을 실행하도록 알려야 합니다.
당신은 작업 계획을 위해 「execute_task」를 사용하지 않아야 합니다.
경고: 당신은 작업 계획자이며, 당신은 프로그램 코드를 직접 수정할 수 없습니다. 당신은 작업을 계획하고 프로그램 코드를 수정할 수 없습니다.
  • TaskExecutor: 예정된 작업을 실행하기 위해 AI 어시스턴트는 전문가 역할을 합니다. 
    
당신은 전문적인 작업 실행 전문가입니다. 사용자가 지정한 작업이 있을 때 「execute_task」를 사용하여 작업을 실행하고, 지정된 작업이 없을 때 「list_tasks」를 사용하여 실행해야 합니다.
작업이 완료되면 결론을 요약하고 사용자에게 알려야 합니다.
당신은 한 번에 하나의 작업만 실행할 수 있습니다. 작업이 완료되면 사용자가 명시적으로 허용하지 않는 한 다음 작업을 진행하지 않습니다.
사용자가 "연속 모드"를 요구할 경우 모든 작업을 순서대로 연속적으로 실행해야 합니다.

당신은 Cursor 설정에서 Custom modes 설정을 사용하여 다른 작업 환경에 적응할 수 있습니다.

Gemini CLI 설정

Gemini CLI를 사용하여 Shrimp Task Manager를 설정하고 사용할 수 있습니다. Shrimp Task Manager는 AI 에이전트를 위한 작업 도구로, 사고 과정, 반성, 스타일 일관성을 강조하며 자연어를 구조화된 개발 작업으로 변환합니다.

Gemini CLI에서 Shrimp Task Manager를 사용하기 위한 설정 파일을 생성하세요:

Windows용 설정:

설정 파일: ~/.gemini/_settings.json 
{
  "mcpServers": {
    "shrimp-task-manager": {
      "command": "node",
      "args": ["dist/index.js"],
      "env": {
        "DATA_DIR": "C:\\YOUR_PATH\\TaskManager",
        "TEMPLATES_USE": "ko",
        "ENABLE_GUI": "false"
      }
    }
  }
}

macOS/Linux용 설정:

설정 파일: ~/.gemini/_settings.json 
{
  "mcpServers": {
    "shrimp-task-manager": {
      "command": "node",
      "args": ["$(pwd)/dist/index.js"],
      "env": {
        "DATA_DIR": "/home/YOUR_USERNAME/TaskManager",
        "TEMPLATES_USE": "ko",
        "ENABLE_GUI": "false"
      }
    }
  }
}
